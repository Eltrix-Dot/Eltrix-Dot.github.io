Most Discord ticket bots lock essential features behind a monthly subscription. Ticketer is built to be the opposite.

✨ What you get

Core capabilities, pulled from the Ticketer docs and command set.

Interactive setup wizard Run /setup to configure channels, ticket category, logs, and staff role via select menus.
Ticket panel + modal flow Users open tickets from a panel button; staff can claim, close, transcript, and manage members.
Transcripts Generate transcripts with /transcript and optionally set a dedicated transcript channel.
Department routing Configure routes using /department_set and move tickets with /move.
Safety controls Blacklist users from opening tickets, plus anti-spam constraints (1 open ticket rule + cooldown after close).
Staff tools Rename tickets, search tickets, view stats, add/remove members, and use anonymous staff responses.

⚙️ Quick Start

From the repository docs (current setup flow uses Python).

1

Clone

git clone https://github.com/Eltrix-Dot/Ticketer.git
cd Ticketer
2

Install dependencies

pip install -r requirements.txt
3

Configure

Create a .env file in the project root:

DISCORD_TOKEN=your_bot_token_here
CLIENT_ID=your_application_client_id
ADMIN_USER_ID=optional_superuser_discord_id
SUPPORT_INVITE_CODE=optional_support_invite
4

Run

python Main.py

🧩 Setup flow (in Discord)

Pulled from the project setup guide.

1) Portal checklist

Enable Server Members Intent. Invite with scopes bot + applications.commands.

2) Run /setup

Owner/superuser only. Select setup channel, ticket category, log channel, and staff role. Confirm deployment.

3) Optional extras

Use /set_transcript for a dedicated transcript channel. Configure department routes for /move.
